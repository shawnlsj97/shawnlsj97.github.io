# Portfolio QA

Use this skill for pre-launch review and regression checks.

## Accessibility

- Semantic HTML.
- Logical heading hierarchy.
- Keyboard navigation.
- Visible focus states.
- Color contrast.
- Useful alt text.
- Accessible names for icon-only links and buttons.
- Motion respects `prefers-reduced-motion`.

## Performance

- Optimized images.
- Sensible build output size.
- Minimal JavaScript.
- No unnecessary dependencies.
- No old Bootstrap, jQuery, Gulp, Travis, or template leftovers.

## SEO And Sharing

- Page title and meta description.
- Canonical URL where appropriate.
- Open Graph metadata.
- Social preview image if available.
- Structured, crawlable content.

## Links And Content

- No broken internal or external links.
- Resume link points to the latest resume.
- Contact links work.
- External links use `rel="noopener noreferrer"` when opening new tabs.
- No confidential Visa information.
- No stale student-portfolio framing.

## Deployment

- GitHub Pages deployment setup is correct.
- Production build succeeds.
- Site works at the deployed base URL.
- Static assets resolve correctly after deployment.
